## What this project shows
This project demonstrates the **Factory Method Design Pattern**, where different user objects 
(Customer, Delivery Partner, Restaurant) are created through a central Factory class.

## Why Factory Method?
- Removes multiple `new` statements
- Centralizes creation
- Easy to extend (add Admin, add Rider, etc.)
- Cleaner, more maintainable code
